# Day 5: Intelligent Prompt Routing

## Objective

Improve the AI Study Mentor by making it intelligently choose different prompts based on the user's question type instead of using a single prompt for every query.

---

## What I Learned

* Learned why a single large prompt is not ideal for every use case.
* Created multiple specialized prompts for different categories.
* Built prompt routing logic using Python.
* Categorized questions into Programming, Comparison, and Factual queries.
* Dynamically selected prompts before sending requests to the AI model.
* Improved AI response quality using Prompt Engineering.
* Understood the concept of Hybrid AI Architecture (Python + LLM).
* Learned how software engineering principles also apply to AI applications.

---

## Challenges Faced

* Python indentation errors while implementing prompt routing.
* Incorrect variable naming (`selected_prompt`).
* Debugging unexpected AI responses.
* Understanding when to use different prompts.
* Improving response quality without making prompts unnecessarily large.

---

## Outcome

Successfully upgraded the AI Study Mentor with intelligent prompt routing, making responses more relevant, modular, and scalable while following better AI Engineering practices.

---

## Key Concepts

* Prompt Engineering
* Prompt Routing
* Hybrid AI Architecture
* Modular Design
* Separation of Concerns
* AI Decision Logic
